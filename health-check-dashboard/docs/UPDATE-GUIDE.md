# 🔄 Обновление данных на следующий месяц

## Шаг 1: Собрать данные

Источники (в порядке приоритета):

| Данные | Источник | Файл |
|--------|----------|------|
| NGR, Deposits, Margin | Margin_Data | `Margin_Data_*.xlsx` → лист GEO_Monthly |
| AU, FTD, ARPPU, Cashout% | Маркетинговые отчёты | `маркетинговый_отчет_*_GEO.xlsx` → лист Export |
| Retention (M1, M3, Core) | MGM.io | user_activity_matrix export |
| Support (FRT, CSAT, AR%, AHT) | BAS | geo_healthcheck export |
| Casino/Sport breakdown | Profit reports | profit_software_games_*.xlsx |

**Важно:** маркетинговые отчёты должны быть за 1 месяц (01.04→30.04), НЕ кумулятивные (01.07→30.04).

## Шаг 2: Обновить `index.html`

Открыть файл в текстовом редакторе. Найти и обновить 4 секции:

### 2.1 Объект `D` (строка ~169)

Заменить значения для каждого GEO. Пример для Brazil:

```js
"🇧🇷 Brazil":{
  au:1297,           // ← новое значение AU
  ftd_count:810,     // ← FTD за месяц
  ngr:81765,         // ← NGR (FIN_Result_USD)
  ngr_mom:40.1,      // ← (NGR_текущий - NGR_прошлый) / |NGR_прошлый| * 100
  dep_approval:86.6, // ← % одобренных депозитов
  // ... остальные метрики
}
```

Если данных нет — ставить `null` (НЕ 0).

### 2.2 Объект `PREV` (строка ~145)

Перенести текущий NGR в PREV. Сейчас:
```js
"🇧🇷 Brazil":{ngr:58365}  // это февраль
```
Заменить на:
```js
"🇧🇷 Brazil":{ngr:81765}  // это март (текущий)
```

### 2.3 Массив `TREND` (строка ~153)

Добавить новый месяц в конец каждого массива:
```js
"🇧🇷 Brazil":[55383,51287,79210,86152,79061,108068,58365,81765,НОВЫЙ],
```

### 2.4 Массив `PLAT_TREND` (строка ~167)

Добавить сумму всех GEO NGR за новый месяц:
```js
const PLAT_TREND = [110262,150972,170247,170487,212263,226268,240102,174723,НОВЫЙ];
```

### 2.5 Массив `TL` (строка ~168)

Добавить label нового месяца:
```js
const TL = ["Aug","Sep","Oct","Nov","Dec","Jan","Feb","Mar","Apr"];
```

## Верификация NGR

**Всегда проверять NGR из 3 источников:**

1. `Margin_Data` → лист `GEO_Monthly` → столбец `FIN_Result_USD`
2. Маркетинговые отчёты → поле `- NGR`
3. `клиенты_*.xlsx` → сумма столбца `NGR`

Все три должны совпадать. Если расхождение > $1,000 — разобраться до коммита.
