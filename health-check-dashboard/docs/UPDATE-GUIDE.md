# 🔄 Обновление данных на следующий месяц

## Новый workflow (v28+)

Данные вынесены из `index.html` в `data.json`. Обновлять можно тремя способами:

### Способ 1: Через браузер (рекомендуется)

1. Открыть дашборд
2. Нажать **📝** (иконка в хедере)
3. Вкладка **🌍 По GEO** — отредактировать метрики для каждого GEO
4. Вкладка **⚙ Мета** — обновить месяц, labels, flags
5. Нажать **⬇ Скачать JSON** — сохранить файл
6. Заменить `data.json` в репозитории
7. Commit + push

### Способ 2: Редактировать data.json напрямую

1. Открыть `data.json` в текстовом редакторе
2. Обновить секции ниже
3. Commit + push

### Способ 3: Загрузить файл

1. Подготовить JSON файл
2. В дашборде нажать **📝** → **📂 Загрузить файл**
3. Данные применяются мгновенно (сохраняются в localStorage)
4. **⬇ Скачать JSON** → заменить `data.json` → commit + push

## Шаг 1: Собрать данные

Источники (в порядке приоритета):

| Данные | Источник | Файл |
|--------|----------|------|
| NGR, Deposits, Margin | Margin_Data | `Margin_Data_*.xlsx` → лист GEO_Monthly |
| AU, FTD, ARPPU, Cashout% | Маркетинговые отчёты | `маркетинговый_отчет_*_GEO.xlsx` → лист Export |
| Retention (M1, M3, Core) | MGM.io | user_activity_matrix export |
| Support (FRT, CSAT, AR%, AHT) | BAS | geo_healthcheck export |
| Casino/Sport breakdown | Profit reports | profit_software_games_*.xlsx |

**Важно:** маркетинговые отчёты должны быть за 1 месяц (01.04→30.04), НЕ кумулятивные (01.07→30.04).

## Шаг 2: Обновить `data.json`

### 2.1 Объект `meta`

```json
{
  "month": "Апрель 2026",
  "month_short": "Apr",
  "prev_month_short": "мар",
  "version": "v28",
  "updated": "2026-05-01"
}
```

### 2.2 Объект `data` — метрики каждого GEO

```json
"🇧🇷 Brazil": {
  "au": 1297,
  "ftd_count": 810,
  "ngr": 81765,
  "ngr_mom": 40.1,
  "dep_approval": 86.6,
  ...
}
```

Если данных нет — ставить `null` (НЕ 0).

### 2.3 Объект `prev`

Перенести текущий NGR в prev:
```json
"🇧🇷 Brazil": {"ngr": 81765}
```

### 2.4 Массив `trend`

Добавить новый месяц в конец каждого массива:
```json
"🇧🇷 Brazil": [55383, 51287, 79210, ..., 81765, НОВЫЙ]
```

### 2.5 `plat_trend` и `trend_labels`

```json
"plat_trend": [110262, 150972, ..., 174723, НОВЫЙ],
"trend_labels": ["Sep","Oct","Nov","Dec","Jan","Feb","Mar","Apr"]
```

### 2.6 `flags`

Если оперативная ситуация изменилась:
```json
{
  "🇵🇪 Peru": {"icon": "⛔", "msg": "Платёжки не работают", "detail": "Все ПС отключены."}
}
```

## Шаг 3: Проверить

1. Открыть дашборд в браузере
2. Проверить платформенный score и NGR
3. Кликнуть на каждый GEO — убедиться что данные обновились
4. Нажать ✈ — проверить Telegram отчёт
5. Commit + push

## Верификация NGR

**Всегда проверять NGR из 3 источников:**

1. `Margin_Data` → лист `GEO_Monthly` → столбец `FIN_Result_USD`
2. Маркетинговые отчёты → поле `- NGR`
3. `клиенты_*.xlsx` → сумма столбца `NGR`

Все три должны совпадать. Если расхождение > $1,000 — разобраться до коммита.
